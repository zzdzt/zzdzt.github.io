Hello World

在看了这么多博客之后，觉得自己也该用博客作为自己记录与分享的地方。总想着自己也建个博客，但总是被建站很难这个想法劝退。随着事情一推再推，距离这个博客想法已经过去两年了。其实这个博客在一年前就建立了，当时我在捣鼓QQ机器人，在查找插件编写教程的时候看到稚昂长的博客，了解到通过Hexo来完成博客搭建，几番摸索之后，把hexo部署到了本地，然后学了点MD语法，最后由于要部署到GitHub上所以先暂时搁置。结果一搁置就是半年过去。

今天————2023年6月14日，在吃饱了撑的情况下，在个人编写的博客项目不够美观（其实各方面功能都不够），在自己服务器部署不理想的前提下，选择重启这个Hexo博客项目.

于是，制杖的紫藤博客搭建成功！（可喜可贺，可喜可贺）

博客搭建过程

如果正在看博客的你也有想要搭建博客的想法，希望我这篇文章能够帮到你。接下来是我部署博客时的一些经验，更建议直接跳转至下方参考资料的文章查看，毕竟我是搬运他们的部分内容过来，咩~~

  • Q：搭建博客需要服务器吗？
    A：不需要，只需要一台可联网的电脑设备（mac\windows\linux）和一个GitHub账户

搭建Hexo博客

Hexo博客框架可以通过npm(node.js的包管理器)安装。安装npm的教程就不在这里列出了，毕竟距离安装的时候已经过去很久，过程几乎忘记了，有需要可以自行百度。

  1. npm安装完成以后，通过在命令行窗口输入命令 npm install hexo-cli -g 安装Hexo
  2. 初始化博客目录 hexo init xxxxx.github.io(xxxxx为你的名字)
  3. 命令行进入新建的博客目录 cd xxxxx.github.io
  4. 通过 hexo clean 清除缓存。通过 hexo g 生成静态页面(g 就是generate 生成的意思)。通过 hexo s 启动本地服务器
  5. 启动本地服务器后，打开浏览器，输入 localhost:4000。你的Hexo博客就搭建完成了.

发表你的第一篇文章

你可以通过以下两种方式来新建你的文章

直接新建文件

在你的博客文件夹根目录下，找到 source 文件夹,在 _posts 新建一篇 文章名.md 的文件，这样你的第一篇博客就建立完成了。
(什么？你问我文件里面填写什么，请听我下文分说)

命令形式

在博客根目录下使用命令行命令 hexo new article（这里的article是你的文章名。执行后会在 source/_posts 下新建一个Markdown文件。和直接新建是一个道理，但我推荐命令形式，比较coooool。

新建博客内容

有了文件之后，编辑你的文件，这里推荐查看官方文档以及查找网上资料，这里篇幅所限，就不介绍如何添加tag和分类了，资料介绍很全面。
以下是我的博客文章头部，title、date为文章标题和发布时间；cover为博客的封面，图片文件可以自定义位置；categories和tags分别为博客文章的分类和标签。然后在分割线 --- 下输入你的markdown格式的博客正文

正式部署

搞定了个人博客，当然要部署到网上让别人访问了，如果没有个人服务器，可以使用 GitHub Pages,借助Github部署自己的博客。

创建GitHub Pages仓库

在GitHub文档上可以得到详细的仓库建立过程。例如我的仓库是wistaria.github.io,到时候我就可以通过 wistaria.github.io 来访问自己的博客。按照教程创建完Github Pages仓库后，你就有自己的仓库地址了。

安装Hexo-deployer-git

如果你已经熟悉如何把自己的代码上传到仓库中，你可以跳过这一步骤。
通过 npm install hexo-deployer-git --save 安装Hexo提供的一键部署功能，让你只需要一条命令就可以把代码上传到Github上。
在开始之前，打开你的 _config.yml 修改参数，在配置最下面的deploy添加,这里的xxx是你仓库的地址，你可以通过code的clone点击获取（HTTPS或者SSH），这里我使用的是SSH作为我的库地址。

推送你的网站到Github上

输入 hexo d （d 是 depoly 的意思），这样你的博客就部署上去了。
如果你的博客进行了修改，记得先输入 hexo g 生成静态文件后再上传，才能提交你的修改。

主题、评论、域名、服务器

主题

如果你觉得博客不够美观，你可以去寻找并安装hexo的博客主题，使你的博客更加美观。

评论

如果你需要评论功能，你也可以去安装评论插件，在许多hexo主题中都会向你介绍如何添加你的评论系统。

域名和服务器

域名可以使朋友更好的记住你的博客地址，如果没有域名，也可以像我一样通过xxxx.github.io来访问我的博客。
服务器相关：因为 GitHub 不让百度爬取索引，也就是说，如果你的朋友想通过百度来搜索你的博客或者你写的文章，是搜索不到的。所以得搞一台自己的服务器，把自己的博客部署到自己的服务器上，这样大家才能认识你
有了域名和服务器，你就完全拥有属于自己的个人网站了
